# Промпт для Claude Code

## Проект: Реестр сертификатов соответствия Тоҷикстандарт

Создай веб-приложение для органа сертификации в Таджикистане. Сотрудники заполняют сертификаты соответствия (Сертификати мутобиқат) и им нужно чтобы данные каждого сертификата автоматически попадали в единую таблицу-реестр.

## Стек

- **Next.js** (App Router)
- **SQLite** через Prisma (или better-sqlite3)
- **Tailwind CSS** для стилей
- **xlsx** (библиотека SheetJS) для экспорта в Excel

## Структура приложения

### Страница 1: Форма ввода сертификата (`/`)

Форма с полями (все подписи на русском языке):

| Поле | Тип | Обязательное |
|------|-----|-------------|
| № п/п | Автоинкремент, не редактируется | — |
| № сертификата соответствия | text | да |
| № копии | text | нет |
| Дата выдачи | date | да |
| Срок действия (до) | date | да |
| Наименование предприятия/организации и адрес | textarea | да |
| Оформление сертификата | select: "1 - Оригинал", "2 - Копия" | да |
| Наименование продукции/товаров | textarea | да |
| Количество (кол-во) | text | да |
| На основании какого документа выдан сертификат | textarea | да |
| Страна происхождения | text (по умолчанию "Тоҷикистон") | да |
| Сумма (столбец L) | number | нет |
| Столбец M | number | нет |
| Столбец N | number | нет |
| Код 1 (столбец O) | text | нет |
| Дата (столбец P) | date | нет |
| Код 2 (столбец Q) | text | нет |

**Важно:** Последние 6 числовых/кодовых полей (столбцы L-Q) — их точное назначение пока неизвестно. Сделай их необязательными с нейтральными подписями. Подписи потом уточним и переименуем.

**Поведение формы:**
- Кнопка "Сохранить в реестр" — сохраняет данные в SQLite
- После сохранения показать уведомление "Запись добавлена" и очистить форму
- Кнопка "Очистить" — сбрасывает форму

### Страница 2: Просмотр реестра (`/registry`)

- Таблица со всеми записями из базы данных
- Столбцы соответствуют полям формы
- Сортировка по дате (новые сверху)
- Поиск по номеру сертификата и названию организации
- Фильтр по месяцу/году (выбор месяца — показывает записи за этот месяц)
- Кнопка **"Скачать Excel"** — генерирует .xlsx файл с заголовком "СВЕДЕНИЯ отдела сертификации продукции за {месяц} {год} года."
- Кнопка **"Удалить"** у каждой строки (с подтверждением)
- Кнопка **"Редактировать"** у каждой строки — открывает форму с заполненными данными

### Страница 3: Приложение к сертификату (`/appendix`) — Замима

Форма для добавления списка продукции к конкретному сертификату:

- Выбор сертификата (по номеру) — dropdown или поиск
- Динамическая таблица:
  - № п/п (автоинкремент)
  - Наименование продукции (text)
  - Единица измерения (text, напр. "кг")
  - Количество (number)
- Кнопка "+ Добавить строку"
- Кнопка "Сохранить приложение"

## Дизайн

- Чистый, профессиональный, минималистичный
- Верхняя панель (header) с названием: **"Реестр сертификатов соответствия"**
- Подзаголовок: **"Агентии Тоҷикстандарт"**
- Цветовая схема: зелёный (#2E7D32 основной) + белый фон — как в шапке их реестра
- Навигация: "Новый сертификат" | "Реестр" | "Приложения"
- Адаптив не критичен (работают с компьютера), но на планшете тоже должно выглядеть нормально

## API Routes (Next.js)

```
POST   /api/certificates      — создать запись
GET    /api/certificates      — получить все записи (с фильтрами ?month=1&year=2025)
PUT    /api/certificates/:id  — обновить запись
DELETE /api/certificates/:id  — удалить запись
GET    /api/certificates/export — скачать Excel файл

POST   /api/appendix          — добавить приложение
GET    /api/appendix/:certId  — получить приложения к сертификату
```

## Формат Excel при экспорте

Первые 2 строки — заголовок:
- Строка 1: "СВЕДЕНИЯ" (объединённая ячейка, по центру)
- Строка 2: "отдела сертификации продукции за {месяц} {год} года." (объединённая ячейка, по центру)
- Строка 3: заголовки столбцов (зелёный фон #90EE90, текст по центру, перенос текста)
- Данные начинаются со строки 4

Столбцы Excel:
A: № п/п
B: № сертификата соответствия  
C: № копии
D: Дата выдачи
E: Срок действия
F: Наименование предприятия/организации и адрес
G: Оформление сертификата
H: Наименование продукции/товаров
I: Кол-во
J: На основании какого документа выдан
K: Страна
L-Q: Дополнительные числовые поля

## База данных (SQLite)

Таблица `certificates`:
- id (PRIMARY KEY, autoincrement)
- cert_number (TEXT, NOT NULL)
- copy_number (TEXT)
- issue_date (TEXT, NOT NULL) 
- expiry_date (TEXT, NOT NULL)
- organization (TEXT, NOT NULL)
- cert_type (INTEGER, DEFAULT 1)
- products (TEXT, NOT NULL)
- quantity (TEXT, NOT NULL)
- basis_document (TEXT, NOT NULL)
- country (TEXT, DEFAULT 'Тоҷикистон')
- col_l (REAL)
- col_m (REAL)
- col_n (REAL)
- col_o (TEXT)
- col_p (TEXT)
- col_q (TEXT)
- created_at (DATETIME, DEFAULT CURRENT_TIMESTAMP)

Таблица `appendixes`:
- id (PRIMARY KEY, autoincrement)
- certificate_id (INTEGER, FOREIGN KEY)
- item_number (INTEGER)
- product_name (TEXT)
- unit (TEXT)
- quantity (REAL)

## Важные детали

1. Интерфейс полностью на **русском языке**
2. Даты в формате **ДД.ММ.ГГ** при отображении
3. После сохранения записи — показать toast/notification "Запись №{номер} добавлена в реестр"
4. Экспорт Excel должен точно воспроизводить формат таблицы клиента (зелёные заголовки, объединённые ячейки в шапке)
5. Приложение должно работать локально (без внешних зависимостей кроме npm пакетов)
